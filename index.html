From Curiosity to Cloud Infrastructure

I started my journey in computer networking with a deep curiosity about how systems communicate. I am the type of person who isn't satisfied with just theory—I love building real projects, troubleshooting complex issues, and designing infrastructure from scratch.

Through continuous learning and competition, I developed strong skills in Linux Server, Network Infrastructure, Virtualization, and Cloud Technology. I actively manage school networks and servers as a core member of Team Computer SMKN 1 Liwa.

My focus is on becoming a Professional Cloud Engineer or Network Engineer. I thrive in environments where I can design enterprise networks, deploy scalable server architectures, and solve critical system problems.

I don't just learn technology; I implement it. Breaking things and fixing them is part of my process. Whether it's configuring VLANs on Mikrotik, deploying Docker containers, or setting up a secure Cloudflare Tunnel, I ensure I understand the underlying protocol, not just the UI.

Work Philosophy

I believe that the best way to learn technology is through hands-on implementation. Every failure is a debug log, and every success is a deployed service. I am highly adaptable, eager to learn new stacks, and obsessed with system optimization and security.

Competing in ITNSA requires mastery of enterprise-level networking, server deployment, and security implementation under strict time limits. It's not just about knowing commands; it's about designing resilient systems.

  • Network Security & Firewall

network_config.rsc
# Mikrotik VLAN & DHCP Setup
/interface vlan add name=vlan10 vlan-id=10 interface=bridge
/interface vlan add name=vlan20 vlan-id=20 interface=bridge

# DHCP Server Configuration
/ip pool add name=pool10 ranges=192.168.10.10-192.168.10.100
/ip dhcp-server add interface=vlan10 address-pool=pool10

# Firewall NAT Rules
/ip firewall nat add chain=srcnat src-address=192.168.10.0/24 action=masquerade
cloudflare_tunnel.sh
#!/bin/bash
# Expose local web server via Cloudflare Tunnel

sudo cloudflared tunnel create ridho_web
sudo cloudflared tunnel route dns ridho_web web.ridho.dev

# Config YAML
tunnel: "ridho_web"
credentials-file: "/root/.cloudflared/ridho_web.json"
ingress:
  - hostname: "web.ridho.dev"
    service: "http://localhost:80"
  - service: "http_status:404"
haproxy.cfg
# HAProxy Load Balancer Configuration
global
  log /dev/log local0
  maxconn 2000

frontend http_front
  bind *:80
  default_backend web_servers

backend web_servers
  balance roundrobin
  server web1 192.168.1.10:80 check
  server web2 192.168.1.11:80 check
